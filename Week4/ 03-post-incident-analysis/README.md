#  Post-Incident Analysis - SOC Project

##  Objective
Conduct post-incident analysis for a phishing attack using Root Cause Analysis (RCA), lessons learned, and SOC metrics.

##  Tools Used
- Google Sheets
- Draw.io

##  Project Components
- Root Cause Analysis (5 Whys)
- Fishbone Diagram (Cause Analysis)
- SOC Metrics (MTTD & MTTR)
- Lessons Learned

##  Incident Summary
A phishing email bypassed email filters and was opened by a user, leading to a malicious link click and potential compromise.



##  Metrics
- Mean Time to Detect (MTTD): 2 hours
- Mean Time to Respond (MTTR): 4 hours



##  Key Findings
- Weak email filtering rules
- Lack of user awareness
- Delayed detection mechanisms



## ✅ Outcome
Improved detection strategy, user training, and SOC response time optimization.
